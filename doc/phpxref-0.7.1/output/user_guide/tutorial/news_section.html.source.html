 426  <h1>News section<a class="headerlink" href="#news-section" title="Permalink to this headline">¶</a></h1>
 427  <p>In the last section, we went over some basic concepts of the framework
 428  by writing a class that includes static pages. We cleaned up the URI by
 429  adding custom routing rules. Now it’s time to introduce dynamic content
 430  and start using a database.</p>
 431  <div class="section" id="setting-up-your-model">
 432  <h2>Setting up your model<a class="headerlink" href="#setting-up-your-model" title="Permalink to this headline">¶</a></h2>
 433  <p>Instead of writing database operations right in the controller, queries
 434  should be placed in a model, so they can easily be reused later. Models
 435  are the place where you retrieve, insert, and update information in your
 436  database or other data stores. They represent your data.</p>
 437  <p>Open up the <em>application/models/</em> directory and create a new file called
 438  <em>News_model.php</em> and add the following code. Make sure you’ve configured
 439  your database properly as described <a class="reference internal" href="../database/configuration.html"><span class="doc">here</span></a>.</p>
 440  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="o">&lt;?</span><span class="nx">php</span>
 441  <span class="k">class</span> <span class="nc">News_model</span> <span class="k">extends</span> <span class="nx">CI_Model</span> <span class="p">{</span>
 442  
 443          <span class="k">public</span> <span class="k">function</span> <span class="nf">__construct</span><span class="p">()</span>
 444          <span class="p">{</span>
 445                  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">database</span><span class="p">();</span>
 446          <span class="p">}</span>
 447  <span class="p">}</span>
 448  </pre></div>
 449  </div>
 450  <p>This code looks similar to the controller code that was used earlier. It
 451  creates a new model by extending <code class="docutils literal"><span class="pre">CI_Model</span></code> and loads the database
 452  library. This will make the database class available through the
 453  <code class="docutils literal"><span class="pre">$this-&gt;db</span></code> object.</p>
 454  <p>Before querying the database, a database schema has to be created.
 455  Connect to your database and run the SQL command below (MySQL).
 456  Also add some seed records.</p>
 457  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nx">CREATE</span> <span class="nx">TABLE</span> <span class="nx">news</span> <span class="p">(</span>
 458          <span class="nx">id</span> <span class="nx">int</span><span class="p">(</span><span class="mi">11</span><span class="p">)</span> <span class="k">NOT</span> <span class="k">NULL</span> <span class="nx">AUTO_INCREMENT</span><span class="p">,</span>
 459          <span class="nx">title</span> <span class="nx">varchar</span><span class="p">(</span><span class="mi">128</span><span class="p">)</span> <span class="k">NOT</span> <span class="k">NULL</span><span class="p">,</span>
 460          <span class="nx">slug</span> <span class="nx">varchar</span><span class="p">(</span><span class="mi">128</span><span class="p">)</span> <span class="k">NOT</span> <span class="k">NULL</span><span class="p">,</span>
 461          <span class="nx">text</span> <span class="nx">text</span> <span class="k">NOT</span> <span class="k">NULL</span><span class="p">,</span>
 462          <span class="nx">PRIMARY</span> <span class="nx">KEY</span> <span class="p">(</span><span class="nx">id</span><span class="p">),</span>
 463          <span class="nx">KEY</span> <span class="nx">slug</span> <span class="p">(</span><span class="nx">slug</span><span class="p">)</span>
 464  <span class="p">);</span>
 465  </pre></div>
 466  </div>
 467  <p>Now that the database and a model have been set up, you’ll need a method
 468  to get all of our posts from our database. To do this, the database
 469  abstraction layer that is included with CodeIgniter —
 470  <a class="reference internal" href="../database/query_builder.html"><span class="doc">Query Builder</span></a> — is used. This makes it
 471  possible to write your ‘queries’ once and make them work on <a class="reference internal" href="../general/requirements.html"><span class="doc">all
 472  supported database systems</span></a>. Add the
 473  following code to your model.</p>
 474  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="k">public</span> <span class="k">function</span> <span class="nf">get_news</span><span class="p">(</span><span class="nv">$slug</span> <span class="o">=</span> <span class="k">FALSE</span><span class="p">)</span>
 475  <span class="p">{</span>
 476          <span class="k">if</span> <span class="p">(</span><span class="nv">$slug</span> <span class="o">===</span> <span class="k">FALSE</span><span class="p">)</span>
 477          <span class="p">{</span>
 478                  <span class="nv">$query</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">db</span><span class="o">-&gt;</span><span class="na">get</span><span class="p">(</span><span class="s1">&#39;news&#39;</span><span class="p">);</span>
 479                  <span class="k">return</span> <span class="nv">$query</span><span class="o">-&gt;</span><span class="na">result_array</span><span class="p">();</span>
 480          <span class="p">}</span>
 481  
 482          <span class="nv">$query</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">db</span><span class="o">-&gt;</span><span class="na">get_where</span><span class="p">(</span><span class="s1">&#39;news&#39;</span><span class="p">,</span> <span class="k">array</span><span class="p">(</span><span class="s1">&#39;slug&#39;</span> <span class="o">=&gt;</span> <span class="nv">$slug</span><span class="p">));</span>
 483          <span class="k">return</span> <span class="nv">$query</span><span class="o">-&gt;</span><span class="na">row_array</span><span class="p">();</span>
 484  <span class="p">}</span>
 485  </pre></div>
 486  </div>
 487  <p>With this code you can perform two different queries. You can get all
 488  news records, or get a news item by its <a class="reference external" href="#">slug</a>. You might have
 489  noticed that the <code class="docutils literal"><span class="pre">$slug</span></code> variable wasn’t sanitized before running the
 490  query; <a class="reference internal" href="../database/query_builder.html"><span class="doc">Query Builder</span></a> does this for you.</p>
 491  </div>
 492  <div class="section" id="display-the-news">
 493  <h2>Display the news<a class="headerlink" href="#display-the-news" title="Permalink to this headline">¶</a></h2>
 494  <p>Now that the queries are written, the model should be tied to the views
 495  that are going to display the news items to the user. This could be done
 496  in our <code class="docutils literal"><span class="pre">Pages</span></code> controller created earlier, but for the sake of clarity,
 497  a new <code class="docutils literal"><span class="pre">News</span></code> controller is defined. Create the new controller at
 498  <em>application/controllers/News.php</em>.</p>
 499  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="o">&lt;?</span><span class="nx">php</span>
 500  <span class="k">class</span> <span class="nc">News</span> <span class="k">extends</span> <span class="nx">CI_Controller</span> <span class="p">{</span>
 501  
 502          <span class="k">public</span> <span class="k">function</span> <span class="nf">__construct</span><span class="p">()</span>
 503          <span class="p">{</span>
 504                  <span class="k">parent</span><span class="o">::</span><span class="na">__construct</span><span class="p">();</span>
 505                  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">model</span><span class="p">(</span><span class="s1">&#39;news_model&#39;</span><span class="p">);</span>
 506                  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">helper</span><span class="p">(</span><span class="s1">&#39;url_helper&#39;</span><span class="p">);</span>
 507          <span class="p">}</span>
 508  
 509          <span class="k">public</span> <span class="k">function</span> <span class="nf">index</span><span class="p">()</span>
 510          <span class="p">{</span>
 511                  <span class="nv">$data</span><span class="p">[</span><span class="s1">&#39;news&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">news_model</span><span class="o">-&gt;</span><span class="na">get_news</span><span class="p">();</span>
 512          <span class="p">}</span>
 513  
 514          <span class="k">public</span> <span class="k">function</span> <span class="nf">view</span><span class="p">(</span><span class="nv">$slug</span> <span class="o">=</span> <span class="k">NULL</span><span class="p">)</span>
 515          <span class="p">{</span>
