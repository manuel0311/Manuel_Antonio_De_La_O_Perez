 426  <h1>Template Parser Class<a class="headerlink" href="#template-parser-class" title="Permalink to this headline">¶</a></h1>
 427  <p>The Template Parser Class can perform simple text substitution for
 428  pseudo-variables contained within your view files.
 429  It can parse simple variables or variable tag pairs.</p>
 430  <p>If you’ve never used a template engine,
 431  pseudo-variable names are enclosed in braces, like this:</p>
 432  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="o">&lt;</span><span class="nx">html</span><span class="o">&gt;</span>
 433          <span class="o">&lt;</span><span class="nx">head</span><span class="o">&gt;</span>
 434                  <span class="o">&lt;</span><span class="nx">title</span><span class="o">&gt;</span><span class="p">{</span><span class="nx">blog_title</span><span class="p">}</span><span class="o">&lt;/</span><span class="nx">title</span><span class="o">&gt;</span>
 435          <span class="o">&lt;/</span><span class="nx">head</span><span class="o">&gt;</span>
 436          <span class="o">&lt;</span><span class="nx">body</span><span class="o">&gt;</span>
 437                  <span class="o">&lt;</span><span class="nx">h3</span><span class="o">&gt;</span><span class="p">{</span><span class="nx">blog_heading</span><span class="p">}</span><span class="o">&lt;/</span><span class="nx">h3</span><span class="o">&gt;</span>
 438  
 439          <span class="p">{</span><span class="nx">blog_entries</span><span class="p">}</span>
 440                  <span class="o">&lt;</span><span class="nx">h5</span><span class="o">&gt;</span><span class="p">{</span><span class="nx">title</span><span class="p">}</span><span class="o">&lt;/</span><span class="nx">h5</span><span class="o">&gt;</span>
 441                  <span class="o">&lt;</span><span class="nx">p</span><span class="o">&gt;</span><span class="p">{</span><span class="nx">body</span><span class="p">}</span><span class="o">&lt;/</span><span class="nx">p</span><span class="o">&gt;</span>
 442          <span class="p">{</span><span class="o">/</span><span class="nx">blog_entries</span><span class="p">}</span>
 443  
 444          <span class="o">&lt;/</span><span class="nx">body</span><span class="o">&gt;</span>
 445  <span class="o">&lt;/</span><span class="nx">html</span><span class="o">&gt;</span>
 446  </pre></div>
 447  </div>
 448  <p>These variables are not actual PHP variables, but rather plain text
 449  representations that allow you to eliminate PHP from your templates
 450  (view files).</p>
 451  <div class="admonition note">
 452  <p class="first admonition-title">Note</p>
 453  <p class="last">CodeIgniter does <strong>not</strong> require you to use this class since
 454  using pure PHP in your view pages lets them run a little faster.
 455  However, some developers prefer to use a template engine if
 456  they work with designers who they feel would find some
 457  confusion working with PHP.</p>
 458  </div>
 459  <div class="admonition important">
 460  <p class="first admonition-title">Important</p>
 461  <p class="last">The Template Parser Class is <strong>not</strong> a full-blown
 462  template parsing solution. We’ve kept it very lean on purpose in order
 463  to maintain maximum performance.</p>
 464  </div>

 479  <h2><a class="toc-backref" href="#id1">Using the Template Parser Class</a><a class="headerlink" href="#using-the-template-parser-class" title="Permalink to this headline">¶</a></h2>
 480  <div class="section" id="initializing-the-class">
 481  <h3><a class="toc-backref" href="#id2">Initializing the Class</a><a class="headerlink" href="#initializing-the-class" title="Permalink to this headline">¶</a></h3>
 482  <p>Like most other classes in CodeIgniter, the Parser class is initialized
 483  in your controller using the <code class="docutils literal"><span class="pre">$this-&gt;load-&gt;library()</span></code> method:</p>
 484  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">library</span><span class="p">(</span><span class="s1">&#39;parser&#39;</span><span class="p">);</span>
 485  </pre></div>
 486  </div>
 487  <p>Once loaded, the Parser library object will be available using:
 488  $this-&gt;parser</p>
 489  </div>
 490  <div class="section" id="parsing-templates">
 491  <h3><a class="toc-backref" href="#id3">Parsing templates</a><a class="headerlink" href="#parsing-templates" title="Permalink to this headline">¶</a></h3>
 492  <p>You can use the <code class="docutils literal"><span class="pre">parse()</span></code> method to parse (or render) simple templates,
 493  like this:</p>
 494  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$data</span> <span class="o">=</span> <span class="k">array</span><span class="p">(</span>
 495          <span class="s1">&#39;blog_title&#39;</span> <span class="o">=&gt;</span> <span class="s1">&#39;My Blog Title&#39;</span><span class="p">,</span>
 496          <span class="s1">&#39;blog_heading&#39;</span> <span class="o">=&gt;</span> <span class="s1">&#39;My Blog Heading&#39;</span>
 497  <span class="p">);</span>
 498  
 499  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">parser</span><span class="o">-&gt;</span><span class="na">parse</span><span class="p">(</span><span class="s1">&#39;blog_template&#39;</span><span class="p">,</span> <span class="nv">$data</span><span class="p">);</span>
 500  </pre></div>
 501  </div>
 502  <p>The first parameter contains the name of the <a class="reference internal" href="../general/views.html"><span class="doc">view
 503  file</span></a> (in this example the file would be called
 504  blog_template.php), and the second parameter contains an associative
 505  array of data to be replaced in the template. In the above example, the
 506  template would contain two variables: {blog_title} and {blog_heading}</p>
 507  <p>There is no need to “echo” or do something with the data returned by
 508  $this-&gt;parser-&gt;parse(). It is automatically passed to the output class
 509  to be sent to the browser. However, if you do want the data returned
 510  instead of sent to the output class you can pass TRUE (boolean) as the
 511  third parameter:</p>
 512  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$string</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">parser</span><span class="o">-&gt;</span><span class="na">parse</span><span class="p">(</span><span class="s1">&#39;blog_template&#39;</span><span class="p">,</span> <span class="nv">$data</span><span class="p">,</span> <span class="k">TRUE</span><span class="p">);</span>
 513  </pre></div>
 514  </div>
 515  </div>
 516  <div class="section" id="variable-pairs">
 517  <h3><a class="toc-backref" href="#id4">Variable Pairs</a><a class="headerlink" href="#variable-pairs" title="Permalink to this headline">¶</a></h3>
 518  <p>The above example code allows simple variables to be replaced. What if
 519  you would like an entire block of variables to be repeated, with each
 520  iteration containing new values? Consider the template example we showed
 521  at the top of the page:</p>
 522  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="o">&lt;</span><span class="nx">html</span><span class="o">&gt;</span>
 523          <span class="o">&lt;</span><span class="nx">head</span><span class="o">&gt;</span>
 524                  <span class="o">&lt;</span><span class="nx">title</span><span class="o">&gt;</span><span class="p">{</span><span class="nx">blog_title</span><span class="p">}</span><span class="o">&lt;/</span><span class="nx">title</span><span class="o">&gt;</span>
 525          <span class="o">&lt;/</span><span class="nx">head</span><span class="o">&gt;</span>
 526          <span class="o">&lt;</span><span class="nx">body</span><span class="o">&gt;</span>
 527                  <span class="o">&lt;</span><span class="nx">h3</span><span class="o">&gt;</span><span class="p">{</span><span class="nx">blog_heading</span><span class="p">}</span><span class="o">&lt;/</span><span class="nx">h3</span><span class="o">&gt;</span>
 528  
 529          <span class="p">{</span><span class="nx">blog_entries</span><span class="p">}</span>
 530                  <span class="o">&lt;</span><span class="nx">h5</span><span class="o">&gt;</span><span class="p">{</span><span class="nx">title</span><span class="p">}</span><span class="o">&lt;/</span><span class="nx">h5</span><span class="o">&gt;</span>
 531                  <span class="o">&lt;</span><span class="nx">p</span><span class="o">&gt;</span><span class="p">{</span><span class="nx">body</span><span class="p">}</span><span class="o">&lt;/</span><span class="nx">p</span><span class="o">&gt;</span>
 532          <span class="p">{</span><span class="o">/</span><span class="nx">blog_entries</span><span class="p">}</span>
 533

 646  your views. It is possible to use a view fragment for what would be inside
