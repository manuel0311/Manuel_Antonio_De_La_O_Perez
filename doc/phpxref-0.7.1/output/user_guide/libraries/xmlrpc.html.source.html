 459  <div class="custom-index container"></div><div class="section" id="what-is-xml-rpc">
 460  <h2><a class="toc-backref" href="#id3">What is XML-RPC?</a><a class="headerlink" href="#what-is-xml-rpc" title="Permalink to this headline">¶</a></h2>
 461  <p>Quite simply it is a way for two computers to communicate over the
 462  internet using XML. One computer, which we will call the client, sends
 463  an XML-RPC <strong>request</strong> to another computer, which we will call the
 464  server. Once the server receives and processes the request it will send
 465  back a <strong>response</strong> to the client.</p>
 466  <p>For example, using the MetaWeblog API, an XML-RPC Client (usually a
 467  desktop publishing tool) will send a request to an XML-RPC Server
 468  running on your site. This request might be a new weblog entry being
 469  sent for publication, or it could be a request for an existing entry for
 470  editing. When the XML-RPC Server receives this request it will examine
 471  it to determine which class/method should be called to process the
 472  request. Once processed, the server will then send back a response
 473  message.</p>
 474  <p>For detailed specifications, you can visit the <a class="reference external" href="http://www.xmlrpc.com/">XML-RPC</a> site.</p>
 475  </div>
 476  <div class="section" id="using-the-xml-rpc-class">
 477  <h2><a class="toc-backref" href="#id4">Using the XML-RPC Class</a><a class="headerlink" href="#using-the-xml-rpc-class" title="Permalink to this headline">¶</a></h2>
 478  <div class="section" id="initializing-the-class">
 479  <h3><a class="toc-backref" href="#id5">Initializing the Class</a><a class="headerlink" href="#initializing-the-class" title="Permalink to this headline">¶</a></h3>
 480  <p>Like most other classes in CodeIgniter, the XML-RPC and XML-RPCS classes
 481  are initialized in your controller using the $this-&gt;load-&gt;library
 482  function:</p>
 483  <p>To load the XML-RPC class you will use:</p>
 484  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">library</span><span class="p">(</span><span class="s1">&#39;xmlrpc&#39;</span><span class="p">);</span>
 485  </pre></div>
 486  </div>
 487  <p>Once loaded, the xml-rpc library object will be available using:
 488  $this-&gt;xmlrpc</p>
 489  <p>To load the XML-RPC Server class you will use:</p>
 490  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">library</span><span class="p">(</span><span class="s1">&#39;xmlrpc&#39;</span><span class="p">);</span>
 491  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">library</span><span class="p">(</span><span class="s1">&#39;xmlrpcs&#39;</span><span class="p">);</span>
 492  </pre></div>
 493  </div>
 494  <p>Once loaded, the xml-rpcs library object will be available using:
 495  $this-&gt;xmlrpcs</p>
 496  <div class="admonition note">
 497  <p class="first admonition-title">Note</p>
 498  <p class="last">When using the XML-RPC Server class you must load BOTH the
 499  XML-RPC class and the XML-RPC Server class.</p>
 500  </div>
 501  </div>
 502  <div class="section" id="sending-xml-rpc-requests">
 503  <h3><a class="toc-backref" href="#id6">Sending XML-RPC Requests</a><a class="headerlink" href="#sending-xml-rpc-requests" title="Permalink to this headline">¶</a></h3>
 504  <p>To send a request to an XML-RPC server you must specify the following
 505  information:</p>
 506  <ul class="simple">
 507  <li>The URL of the server</li>
 508  <li>The method on the server you wish to call</li>
 509  <li>The <em>request</em> data (explained below).</li>
 510  </ul>
 511  <p>Here is a basic example that sends a simple Weblogs.com ping to the
 512  <a class="reference external" href="http://pingomatic.com/">Ping-o-Matic</a></p>
 513  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">library</span><span class="p">(</span><span class="s1">&#39;xmlrpc&#39;</span><span class="p">);</span>
 514  
 515  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">xmlrpc</span><span class="o">-&gt;</span><span class="na">server</span><span class="p">(</span><span class="s1">&#39;http://rpc.pingomatic.com/&#39;</span><span class="p">,</span> <span class="mi">80</span><span class="p">);</span>
 516  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">xmlrpc</span><span class="o">-&gt;</span><span class="na">method</span><span class="p">(</span><span class="s1">&#39;weblogUpdates.ping&#39;</span><span class="p">);</span>
 517  
 518  <span class="nv">$request</span> <span class="o">=</span> <span class="k">array</span><span class="p">(</span><span class="s1">&#39;My Photoblog&#39;</span><span class="p">,</span> <span class="s1">&#39;http://www.my-site.com/photoblog/&#39;</span><span class="p">);</span>
 519  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">xmlrpc</span><span class="o">-&gt;</span><span class="na">request</span><span class="p">(</span><span class="nv">$request</span><span class="p">);</span>
 520  
 521  <span class="k">if</span> <span class="p">(</span> <span class="o">!</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">xmlrpc</span><span class="o">-&gt;</span><span class="na">send_request</span><span class="p">())</span>
 522  <span class="p">{</span>
 523          <span class="k">echo</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">xmlrpc</span><span class="o">-&gt;</span><span class="na">display_error</span><span class="p">();</span>
 524  <span class="p">}</span>
 525  </pre></div>
 526  </div>
 527  <div class="section" id="explanation">
 528  <h4><a class="toc-backref" href="#id7">Explanation</a><a class="headerlink" href="#explanation" title="Permalink to this headline">¶</a></h4>
 529  <p>The above code initializes the XML-RPC class, sets the server URL and
 530  method to be called (weblogUpdates.ping). The request (in this case, the
 531  title and URL of your site) is placed into an array for transportation,
 532  and compiled using the request() function. Lastly, the full request is
 533  sent. If the send_request() method returns false we will display the
 534  error message sent back from the XML-RPC Server.</p>
 535  </div>
 536  </div>
 537  <div class="section" id="anatomy-of-a-request">
 538  <h3><a class="toc-backref" href="#id8">Anatomy of a Request</a><a class="headerlink" href="#anatomy-of-a-request" title="Permalink to this headline">¶</a></h3>
 539  <p>An XML-RPC request is simply the data you are sending to the XML-RPC
 540  server. Each piece of data in a request is referred to as a request
 541  parameter. The above example has two parameters: The URL and title of
 542  your site. When the XML-RPC server receives your request, it will look
 543  for parameters it requires.</p>
 544  <p>Request parameters must be placed into an array for transportation, and
 545  each parameter can be one of seven data types (strings, numbers, dates,
 546  etc.). If your parameters are something other than strings you will have
 547  to include the data type in the request array.</p>
 548  <p>Here is an example of a simple array with three parameters:</p>
 549  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$request</span> <span class="o">=</span> <span class="k">array</span><span class="p">(</span><span class="s1">&#39;John&#39;</span><span class="p">,</span> <span class="s1">&#39;Doe&#39;</span><span class="p">,</span> <span class="s1">&#39;www.some-site.com&#39;</span><span class="p">);</span>
 550  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">xmlrpc</span><span class="o">-&gt;</span><span class="na">request</span><span class="p">(</span><span class="nv">$request</span><span class="p">);</span>
 551  </pre></div>
 552  </div>
 553  <p>If you use data types other than strings, or if you have several
 554  different data types, you will place each parameter into its own array,
 555  with the data type in the second position:</p>
 556  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$request</span> <span class="o">=</span> <span class="k">array</span><span class="p">(</span>
 557          <span class="k">array</span><span class="p">(</span><span class="s1">&#39;John&#39;</span><span class="p">,</span> <span class="s1">&#39;string&#39;</span><span class="p">),</span>
 558          <span class="k">array</span><span class="p">(</span><span class="s1">&#39;Doe&#39;</span><span class="p">,</span> <span class="s1">&#39;string&#39;</span><span class="p">),</span>
 559          <span class="k">array</span><span class="p">(</span><span class="k">FALSE</span><span class="p">,</span> <span class="s1">&#39;boolean&#39;</span><span class="p">),</span>
 560          <span class="k">array</span><span class="p">(</span><span class="mi">12345</span><span class="p">,</span> <span class="s1">&#39;int&#39;</span><span class="p">)</span>
 561  <span class="p">);</span>
 562  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">xmlrpc</span><span class="o">-&gt;</span><span class="na">request</span><span class="p">(</span><span class="nv">$request</span><span class="p">);</span>
 563  </pre></div>
 564  </div>
 565  <p>The <a class="reference external" href="#datatypes">Data Types</a> section below has a full list of data
 566  types.</p>
 567  </div>
 568  <div class="section" id="creating-an-xml-rpc-server">
 569  <h3><a class="toc-backref" href="#id9">Creating an XML-RPC Server</a><a class="headerlink" href="#creating-an-xml-rpc-server" title="Permalink to this headline">¶</a></h3>
 570  <p>An XML-RPC Server acts as a traffic cop of sorts, waiting for incoming
 571  requests and redirecting them to the appropriate functions for
 572  processing.</p>
 573  <p>To create your own XML-RPC server involves initializing the XML-RPC
 574  Server class in your controller where you expect the incoming request to
 575  appear, then setting up an array with mapping instructions so that
 576  incoming requests can be sent to the appropriate class and method for
 577  processing.</p>
 578  <p>Here is an example to illustrate:</p>
 579  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">library</span><span class="p">(</span><span class="s1">&#39;xmlrpc&#39;</span><span class="p">);</span>
 580  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">load</span><span class="o">-&gt;</span><span class="na">library</span><span class="p">(</span><span class="s1">&#39;xmlrpcs&#39;</span><span class="p">);</span>
 581  
 582  <span class="nv">$config</span><span class="p">[</span><span class="s1">&#39;functions&#39;</span><span class="p">][</span><span class="s1">&#39;new_post&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="k">array</span><span class="p">(</span><span class="s1">&#39;function&#39;</span> <span class="o">=&gt;</span> <span class="s1">&#39;My_blog.new_entry&#39;</span><span class="p">);</span>
 583  <span class="nv">$config</span><span class="p">[</span><span class="s1">&#39;functions&#39;</span><span class="p">][</span><span class="s1">&#39;update_post&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="k">array</span><span class="p">(</span><span class="s1">&#39;function&#39;</span> <span class="o">=&gt;</span> <span class="s1">&#39;My_blog.update_entry&#39;</span><span class="p">);</span>
 584  <span class="nv">$config</span><span class="p">[</span><span class="s1">&#39;object&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="nv">$this</span><span class="p">;</span>
 585  
 586  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">xmlrpcs</span><span class="o">-&gt;</span><span class="na">initialize</span><span class="p">(</span><span class="nv">$config</span><span class="p">);</span>
 587  <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">xmlrpcs</span><span class="o">-&gt;</span><span class="na">serve</span><span class="p">();</span>
 588  </pre></div>
 589  </div>
 590  <p>The above example contains an array specifying two method requests that
 591  the Server allows. The allowed methods are on the left side of the
 592  array. When either of those are received, they will be mapped to the
 593  class and method on the right.</p>
 594  <p>The ‘object’ key is a special key that you pass an instantiated class
 595  object with, which is necessary when the method you are mapping to is
 596  not part of the CodeIgniter super object.</p>
 597  <p>In other words, if an XML-RPC Client sends a request for the new_post
 598  method, your server will load the My_blog class and call the new_entry
 599  function. If the request is for the update_post method, your server
 600  will load the My_blog class and call the <code class="docutils literal"><span class="pre">update_entry()</span></code> method.</p>
 601  <p>The function names in the above example are arbitrary. You’ll decide
 602  what they should be called on your server, or if you are using
 603  standardized APIs, like the Blogger or MetaWeblog API, you’ll use their
 604  function names.</p>
 605  <p>There are two additional configuration keys you may make use of when
 606  initializing the server class: debug can be set to TRUE in order to
 607  enable debugging, and xss_clean may be set to FALSE to prevent sending
 608  data through the Security library’s <code class="docutils literal"><span class="pre">xss_clean()</span></code> method.</p>
 609  </div>

 612  <p>When the XML-RPC Server receives a request and loads the class/method
 613  for processing, it will pass an object to that method containing the
 614  data sent by the client.</p>
 615  <p>Using the above example, if the new_post method is requested, the
 616  server will expect a class to exist with this prototype:</p>
 617  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="k">class</span> <span class="nc">My_blog</span> <span class="k">extends</span> <span class="nx">CI_Controller</span> <span class="p">{</span>
 618  
 619          <span class="k">public</span> <span class="k">function</span> <span class="nf">new_post</span><span class="p">(</span><span class="nv">$request</span><span class="p">)</span>

 621  
 622          <span class="p">}</span>
 623  <span class="p">}</span>
 624  </pre></div>
 625  </div>
 626  <p>The $request variable is an object compiled by the Server, which
 627  contains the data sent by the XML-RPC Client. Using this object you will

 630  <p>Below is a real-world example, using the Blogger API. One of the methods
 631  in the Blogger API is <code class="docutils literal"><span class="pre">getUserInfo()</span></code>. Using this method, an XML-RPC

 633  sends back information about that particular user (nickname, user ID,
 634  email address, etc.). Here is how the processing function might look:</p>
 635  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="k">class</span> <span class="nc">My_blog</span> <span class="k">extends</span> <span class="nx">CI_Controller</span> <span class="p">{</span>
 636

 645

 670  <p>The <code class="docutils literal"><span class="pre">output_parameters()</span></code> method retrieves an indexed array
 671  corresponding to the request parameters sent by the client. In the above
 672  example, the output parameters will be the username and password.</p>

 679  <div class="section" id="formatting-a-response">
 680  <h3><a class="toc-backref" href="#id12">Formatting a Response</a><a class="headerlink" href="#formatting-a-response" title="Permalink to this headline">¶</a></h3>
 681  <p>Similar to <em>Requests</em>, <em>Responses</em> must be formatted as an array.
 682  However, unlike requests, a response is an array <strong>that contains a
 683  single item</strong>. This item can be an array with several additional arrays,
 684  but there can be only one primary array index. In other words, the basic
 685  prototype is this:</p>
 686  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$response</span> <span class="o">=</span> <span class="k">array</span><span class="p">(</span><span class="s1">&#39;Response data&#39;</span><span class="p">,</span> <span class="s1">&#39;array&#39;</span><span class="p">);</span>
 687  </pre></div>
 688  </div>
 689  <p>Responses, however, usually contain multiple pieces of information. In
 690  order to accomplish this we must put the response into its own array so
 691  that the primary array continues to contain a single piece of data.
 692  Here’s an example showing how this might be accomplished:</p>
 693  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$response</span> <span class="o">=</span> <span class="k">array</span><span class="p">(</span>
 694          <span class="k">array</span><span class="p">(</span>
 695                  <span class="s1">&#39;first_name&#39;</span> <span class="o">=&gt;</span> <span class="k">array</span><span class="p">(</span><span class="s1">&#39;John&#39;</span><span class="p">,</span> <span class="s1">&#39;string&#39;</span><span class="p">),</span>
 696                  <span class="s1">&#39;last_name&#39;</span> <span class="o">=&gt;</span> <span class="k">array</span><span class="p">(</span><span class="s1">&#39;Doe&#39;</span><span class="p">,</span> <span class="s1">&#39;string&#39;</span><span class="p">),</span>
 697                  <span class="s1">&#39;member_id&#39;</span> <span class="o">=&gt;</span> <span class="k">array</span><span class="p">(</span><span class="mi">123435</span><span class="p">,</span> <span class="s1">&#39;int&#39;</span><span class="p">),</span>
 698                  <span class="s1">&#39;todo_list&#39;</span> <span class="o">=&gt;</span> <span class="k">array</span><span class="p">(</span><span class="k">array</span><span class="p">(</span><span class="s1">&#39;clean house&#39;</span><span class="p">,</span> <span class="s1">&#39;call mom&#39;</span><span class="p">,</span> <span class="s1">&#39;water plants&#39;</span><span class="p">),</span> <span class="s1">&#39;array&#39;</span><span class="p">),</span>
 699          <span class="p">),</span>
 700          <span class="s1">&#39;struct&#39;</span>
 701  <span class="p">);</span>
 702  </pre></div>
 703  </div>
 704  <p>Notice that the above array is formatted as a struct. This is the most
 705  common data type for responses.</p>
 706  <p>As with Requests, a response can be one of the seven data types listed
 707  in the <a class="reference external" href="#datatypes">Data Types</a> section.</p>
 708  </div>

 768  <h4><a class="toc-backref" href="#id16">The Server</a><a class="headerlink" href="#the-server" title="Permalink to this headline">¶</a></h4>
