 426  <h1>URI Class<a class="headerlink" href="#uri-class" title="Permalink to this headline">¶</a></h1>
 427  <p>The URI Class provides methods that help you retrieve information from
 428  your URI strings. If you use URI routing, you can also retrieve
 429  information about the re-routed segments.</p>
 430  <div class="admonition note">
 431  <p class="first admonition-title">Note</p>
 432  <p class="last">This class is initialized automatically by the system so there
 433  is no need to do it manually.</p>
 434  </div>

 445  <dd><dl class="method">
 446  <dt id="CI_URI::segment">
 447  <code class="descname">segment</code><span class="sig-paren">(</span><em>$n</em><span class="optional">[</span>, <em>$no_result = NULL</em><span class="optional">]</span><span class="sig-paren">)</span><a class="headerlink" href="#CI_URI::segment" title="Permalink to this definition">¶</a></dt>
 448  <dd><table class="docutils field-list" frame="void" rules="none">
 449  <col class="field-name" />
 450  <col class="field-body" />
 451  <tbody valign="top">
 452  <tr class="field-odd field"><th class="field-name">Parameters:</th><td class="field-body"><ul class="first simple">
 453  <li><strong>$n</strong> (<em>int</em>) – Segment index number</li>
 454  <li><strong>$no_result</strong> (<em>mixed</em>) – What to return if the searched segment is not found</li>
 455  </ul>
 456  </td>
 457  </tr>
 458  <tr class="field-even field"><th class="field-name">Returns:</th><td class="field-body"><p class="first">Segment value or $no_result value if not found</p>
 459  </td>
 460  </tr>
 461  <tr class="field-odd field"><th class="field-name">Return type:</th><td class="field-body"><p class="first last">mixed</p>
 462  </td>
 463  </tr>
 464  </tbody>
 465  </table>
 466  <p>Permits you to retrieve a specific segment. Where n is the segment
 467  number you wish to retrieve. Segments are numbered from left to right.
 468  For example, if your full URL is this:</p>
 469  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nx">http</span><span class="o">://</span><span class="nx">example</span><span class="o">.</span><span class="nx">com</span><span class="o">/</span><span class="nx">index</span><span class="o">.</span><span class="nx">php</span><span class="o">/</span><span class="nx">news</span><span class="o">/</span><span class="nx">local</span><span class="o">/</span><span class="nx">metro</span><span class="o">/</span><span class="nx">crime_is_up</span>
 470  </pre></div>
 471  </div>
 472  <p>The segment numbers would be this:</p>
 473  <ol class="arabic simple">
 474  <li>news</li>
 475  <li>local</li>
 476  <li>metro</li>
 477  <li>crime_is_up</li>
 478  </ol>
 479  <p>The optional second parameter defaults to NULL and allows you to set the return value
 480  of this method when the requested URI segment is missing.
 481  For example, this would tell the method to return the number zero in the event of failure:</p>
 482  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$product_id</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">uri</span><span class="o">-&gt;</span><span class="na">segment</span><span class="p">(</span><span class="mi">3</span><span class="p">,</span> <span class="mi">0</span><span class="p">);</span>
 483  </pre></div>
 484  </div>
 485  <p>It helps avoid having to write code like this:</p>
 486  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="k">if</span> <span class="p">(</span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">uri</span><span class="o">-&gt;</span><span class="na">segment</span><span class="p">(</span><span class="mi">3</span><span class="p">)</span> <span class="o">===</span> <span class="k">FALSE</span><span class="p">)</span>
 487  <span class="p">{</span>
 488          <span class="nv">$product_id</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span>
 489  <span class="p">}</span>
 490  <span class="k">else</span>
 491  <span class="p">{</span>
 492          <span class="nv">$product_id</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">uri</span><span class="o">-&gt;</span><span class="na">segment</span><span class="p">(</span><span class="mi">3</span><span class="p">);</span>
 493  <span class="p">}</span>
 494  </pre></div>
 495  </div>
 496  </dd></dl>

 609  <p>This method lets you turn URI segments into an associative array of
 610  key/value pairs. Consider this URI:</p>
 611  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nx">index</span><span class="o">.</span><span class="nx">php</span><span class="o">/</span><span class="nx">user</span><span class="o">/</span><span class="nx">search</span><span class="o">/</span><span class="nx">name</span><span class="o">/</span><span class="nx">joe</span><span class="o">/</span><span class="nx">location</span><span class="o">/</span><span class="nx">UK</span><span class="o">/</span><span class="nx">gender</span><span class="o">/</span><span class="nx">male</span>
 612  </pre></div>
 613  </div>
 614  <p>Using this method you can turn the URI into an associative array with
 615  this prototype:</p>
 616  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="p">[</span><span class="k">array</span><span class="p">]</span>
 617  <span class="p">(</span>
 618          <span class="s1">&#39;name&#39;</span>          <span class="o">=&gt;</span> <span class="s1">&#39;joe&#39;</span>
 619          <span class="s1">&#39;location&#39;</span>      <span class="o">=&gt;</span> <span class="s1">&#39;UK&#39;</span>
 620          <span class="s1">&#39;gender&#39;</span>        <span class="o">=&gt;</span> <span class="s1">&#39;male&#39;</span>
 621  <span class="p">)</span>
 622  </pre></div>
 623  </div>
 624  <p>The first parameter lets you set an offset, which defaults to 3 since your
 625  URI will normally contain a controller/method pair in the first and second segments.
 626  Example:</p>
 627  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$array</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">uri</span><span class="o">-&gt;</span><span class="na">uri_to_assoc</span><span class="p">(</span><span class="mi">3</span><span class="p">);</span>
 628  <span class="k">echo</span> <span class="nv">$array</span><span class="p">[</span><span class="s1">&#39;name&#39;</span><span class="p">];</span>
 629  </pre></div>
 630  </div>
 631  <p>The second parameter lets you set default key names, so that the array
 632  returned will always contain expected indexes, even if missing from the URI.
 633  Example:</p>
 634  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$default</span> <span class="o">=</span> <span class="k">array</span><span class="p">(</span><span class="s1">&#39;name&#39;</span><span class="p">,</span> <span class="s1">&#39;gender&#39;</span><span class="p">,</span> <span class="s1">&#39;location&#39;</span><span class="p">,</span> <span class="s1">&#39;type&#39;</span><span class="p">,</span> <span class="s1">&#39;sort&#39;</span><span class="p">);</span>
 635  <span class="nv">$array</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">uri</span><span class="o">-&gt;</span><span class="na">uri_to_assoc</span><span class="p">(</span><span class="mi">3</span><span class="p">,</span> <span class="nv">$default</span><span class="p">);</span>
 636  </pre></div>
 637  </div>
 638  <p>If the URI does not contain a value in your default, an array index will
 639  be set to that name, with a value of NULL.</p>
 640  <p>Lastly, if a corresponding value is not found for a given key (if there
 641  is an odd number of URI segments) the value will be set to NULL.</p>
