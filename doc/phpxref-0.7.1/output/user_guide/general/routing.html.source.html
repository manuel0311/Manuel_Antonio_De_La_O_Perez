 425    <div class="section" id="uri-routing">
 426  <h1>URI Routing<a class="headerlink" href="#uri-routing" title="Permalink to this headline">¶</a></h1>
 427  <p>Typically there is a one-to-one relationship between a URL string and
 428  its corresponding controller class/method. The segments in a URI
 429  normally follow this pattern:</p>
 430  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nx">example</span><span class="o">.</span><span class="nx">com</span><span class="o">/</span><span class="nx">class</span><span class="o">/</span><span class="nx">function</span><span class="o">/</span><span class="nx">id</span><span class="o">/</span>
 431  </pre></div>
 432  </div>
 433  <p>In some instances, however, you may want to remap this relationship so
 434  that a different class/method can be called instead of the one
 435  corresponding to the URL.</p>
 436  <p>For example, let’s say you want your URLs to have this prototype:</p>
 437  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nx">example</span><span class="o">.</span><span class="nx">com</span><span class="o">/</span><span class="nx">product</span><span class="o">/</span><span class="mi">1</span><span class="o">/</span>
 438  <span class="nx">example</span><span class="o">.</span><span class="nx">com</span><span class="o">/</span><span class="nx">product</span><span class="o">/</span><span class="mi">2</span><span class="o">/</span>
 439  <span class="nx">example</span><span class="o">.</span><span class="nx">com</span><span class="o">/</span><span class="nx">product</span><span class="o">/</span><span class="mi">3</span><span class="o">/</span>
 440  <span class="nx">example</span><span class="o">.</span><span class="nx">com</span><span class="o">/</span><span class="nx">product</span><span class="o">/</span><span class="mi">4</span><span class="o">/</span>
 441  </pre></div>
 442  </div>
 443  <p>Normally the second segment of the URL is reserved for the method
 444  name, but in the example above it instead has a product ID. To
 445  overcome this, CodeIgniter allows you to remap the URI handler.</p>
 446  <div class="section" id="setting-your-own-routing-rules">
 447  <h2>Setting your own routing rules<a class="headerlink" href="#setting-your-own-routing-rules" title="Permalink to this headline">¶</a></h2>
 448  <p>Routing rules are defined in your <em>application/config/routes.php</em> file.
 449  In it you’ll see an array called <code class="docutils literal"><span class="pre">$route</span></code> that permits you to specify
 450  your own routing criteria. Routes can either be specified using wildcards
 451  or Regular Expressions.</p>
 452  </div>
 453  <div class="section" id="wildcards">
 454  <h2>Wildcards<a class="headerlink" href="#wildcards" title="Permalink to this headline">¶</a></h2>
 455  <p>A typical wildcard route might look something like this:</p>
 456  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;product/:num&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;catalog/product_lookup&#39;</span><span class="p">;</span>
 457  </pre></div>
 458  </div>
 459  <p>In a route, the array key contains the URI to be matched, while the
 460  array value contains the destination it should be re-routed to. In the
 461  above example, if the literal word “product” is found in the first
 462  segment of the URL, and a number is found in the second segment, the
 463  “catalog” class and the “product_lookup” method are instead used.</p>
 464  <p>You can match literal values or you can use two wildcard types:</p>
 465  <p><strong>(:num)</strong> will match a segment containing only numbers.
 466  <strong>(:any)</strong> will match a segment containing any character (except for ‘/’, which is the segment delimiter).</p>
 467  <div class="admonition note">
 468  <p class="first admonition-title">Note</p>
 469  <p class="last">Wildcards are actually aliases for regular expressions, with
 470  <strong>:any</strong> being translated to <strong>[^/]+</strong> and <strong>:num</strong> to <strong>[0-9]+</strong>,
 471  respectively.</p>
 472  </div>
 473  <div class="admonition note">
 474  <p class="first admonition-title">Note</p>
 475  <p class="last">Routes will run in the order they are defined. Higher routes
 476  will always take precedence over lower ones.</p>
 477  </div>
 478  <div class="admonition note">
 479  <p class="first admonition-title">Note</p>
 480  <p class="last">Route rules are not filters! Setting a rule of e.g.
 481  ‘foo/bar/(:num)’ will not prevent controller <em>Foo</em> and method
 482  <em>bar</em> to be called with a non-numeric value if that is a valid
 483  route.</p>
 484  </div>
 485  </div>
 486  <div class="section" id="examples">
 487  <h2>Examples<a class="headerlink" href="#examples" title="Permalink to this headline">¶</a></h2>
 488  <p>Here are a few routing examples:</p>
 489  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;journals&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;blogs&#39;</span><span class="p">;</span>
 490  </pre></div>
 491  </div>
 492  <p>A URL containing the word “journals” in the first segment will be
 493  remapped to the “blogs” class.</p>
 494  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;blog/joe&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;blogs/users/34&#39;</span><span class="p">;</span>
 495  </pre></div>
 496  </div>
 497  <p>A URL containing the segments blog/joe will be remapped to the “blogs”
 498  class and the “users” method. The ID will be set to “34”.</p>
 499  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;product/(:any)&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;catalog/product_lookup&#39;</span><span class="p">;</span>
 500  </pre></div>
 501  </div>
 502  <p>A URL with “product” as the first segment, and anything in the second
 503  will be remapped to the “catalog” class and the “product_lookup”
 504  method.</p>
 505  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;product/(:num)&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;catalog/product_lookup_by_id/$1&#39;</span><span class="p">;</span>
 506  </pre></div>
 507  </div>
 508  <p>A URL with “product” as the first segment, and a number in the second
 509  will be remapped to the “catalog” class and the
 510  “product_lookup_by_id” method passing in the match as a variable to
 511  the method.</p>
 512  <div class="admonition important">
 513  <p class="first admonition-title">Important</p>
 514  <p class="last">Do not use leading/trailing slashes.</p>
 515  </div>
 516  </div>
 517  <div class="section" id="regular-expressions">
 518  <h2>Regular Expressions<a class="headerlink" href="#regular-expressions" title="Permalink to this headline">¶</a></h2>
 519  <p>If you prefer you can use regular expressions to define your routing
 520  rules. Any valid regular expression is allowed, as are back-references.</p>
 521  <div class="admonition note">
 522  <p class="first admonition-title">Note</p>
 523  <p class="last">If you use back-references you must use the dollar syntax
 524  rather than the double backslash syntax.</p>
 525  </div>
 526  <p>A typical RegEx route might look something like this:</p>
 527  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;products/([a-z]+)/(\d+)&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;$1/id_$2&#39;</span><span class="p">;</span>
 528  </pre></div>
 529  </div>
 530  <p>In the above example, a URI similar to products/shirts/123 would instead
 531  call the “shirts” controller class and the “id_123” method.</p>
 532  <p>With regular expressions, you can also catch multiple segments at once.
 533  For example, if a user accesses a password protected area of your web
 534  application and you wish to be able to redirect them back to the same
 535  page after they log in, you may find this example useful:</p>
 536  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;login/(.+)&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;auth/login/$1&#39;</span><span class="p">;</span>
 537  </pre></div>
 538  </div>
 539  <div class="admonition note">
 540  <p class="first admonition-title">Note</p>
 541  <p class="last">In the above example, if the <code class="docutils literal"><span class="pre">$1</span></code> placeholder contains a
 542  slash, it will still be split into multiple parameters when
 543  passed to <code class="docutils literal"><span class="pre">Auth::login()</span></code>.</p>
 544  </div>
 545  <p>For those of you who don’t know regular expressions and want to learn
 546  more about them, <a class="reference external" href="http://www.regular-expressions.info/">regular-expressions.info</a>
 547  might be a good starting point.</p>
 548  <div class="admonition note">
 549  <p class="first admonition-title">Note</p>
 550  <p class="last">You can also mix and match wildcards with regular expressions.</p>
 551  </div>
 552  </div>
 553  <div class="section" id="callbacks">
 554  <h2>Callbacks<a class="headerlink" href="#callbacks" title="Permalink to this headline">¶</a></h2>
 555  <p>You can also use callbacks in place of the normal routing rules to process
 556  the back-references. Example:</p>
 557  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;products/([a-zA-Z]+)/edit/(\d+)&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="k">function</span> <span class="p">(</span><span class="nv">$product_type</span><span class="p">,</span> <span class="nv">$id</span><span class="p">)</span>
 558  <span class="p">{</span>
 559          <span class="k">return</span> <span class="s1">&#39;catalog/product_edit/&#39;</span> <span class="o">.</span> <span class="nb">strtolower</span><span class="p">(</span><span class="nv">$product_type</span><span class="p">)</span> <span class="o">.</span> <span class="s1">&#39;/&#39;</span> <span class="o">.</span> <span class="nv">$id</span><span class="p">;</span>
 560  <span class="p">};</span>
 561  </pre></div>
 562  </div>
 563  </div>
 564  <div class="section" id="using-http-verbs-in-routes">
 565  <h2>Using HTTP verbs in routes<a class="headerlink" href="#using-http-verbs-in-routes" title="Permalink to this headline">¶</a></h2>
 566  <p>It is possible to use HTTP verbs (request method) to define your routing rules.
 567  This is particularly useful when building RESTful applications. You can use standard HTTP
 568  verbs (GET, PUT, POST, DELETE, PATCH) or a custom one such (e.g. PURGE). HTTP verb rules
 569  are case-insensitive. All you need to do is to add the verb as an array key to your route.
 570  Example:</p>
 571  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;products&#39;</span><span class="p">][</span><span class="s1">&#39;put&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;product/insert&#39;</span><span class="p">;</span>
 572  </pre></div>
 573  </div>
 574  <p>In the above example, a PUT request to URI “products” would call the <code class="docutils literal"><span class="pre">Product::insert()</span></code>
 575  controller method.</p>
 576  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;products/(:num)&#39;</span><span class="p">][</span><span class="s1">&#39;DELETE&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;product/delete/$1&#39;</span><span class="p">;</span>
 577  </pre></div>
 578  </div>
 579  <p>A DELETE request to URL with “products” as first the segment and a number in the second will be
 580  mapped to the <code class="docutils literal"><span class="pre">Product::delete()</span></code> method, passing the numeric value as the first parameter.</p>
 581  <p>Using HTTP verbs is of course, optional.</p>
 582  </div>
 583  <div class="section" id="reserved-routes">
 584  <h2>Reserved Routes<a class="headerlink" href="#reserved-routes" title="Permalink to this headline">¶</a></h2>
 585  <p>There are three reserved routes:</p>
 586  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;default_controller&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;welcome&#39;</span><span class="p">;</span>
 587  </pre></div>
 588  </div>
 589  <p>This route points to the action that should be executed if the URI contains
 590  no data, which will be the case when people load your root URL.
 591  The setting accepts a <strong>controller/method</strong> value and <code class="docutils literal"><span class="pre">index()</span></code> would be
 592  the default method if you don’t specify one. In the above example, it is
 593  <code class="docutils literal"><span class="pre">Welcome::index()</span></code> that would be called.</p>
 594  <div class="admonition note">
 595  <p class="first admonition-title">Note</p>
 596  <p class="last">You can NOT use a directory as a part of this setting!</p>
 597  </div>
 598  <p>You are encouraged to always have a default route as otherwise a 404 page
 599  will appear by default.</p>
 600  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;404_override&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;&#39;</span><span class="p">;</span>
 601  </pre></div>
 602  </div>
 603  <p>This route indicates which controller class should be loaded if the
 604  requested controller is not found. It will override the default 404
 605  error page. Same per-directory rules as with ‘default_controller’
 606  apply here as well.</p>
 607  <p>It won’t affect to the <code class="docutils literal"><span class="pre">show_404()</span></code> function, which will
 608  continue loading the default <em>error_404.php</em> file at
 609  <em>application/views/errors/error_404.php</em>.</p>
 610  <div class="highlight-ci"><div class="highlight"><pre><span></span><span class="nv">$route</span><span class="p">[</span><span class="s1">&#39;translate_uri_dashes&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="k">FALSE</span><span class="p">;</span>
 611  </pre></div>
 612  </div>
 613  <p>As evident by the boolean value, this is not exactly a route. This
 614  option enables you to automatically replace dashes (‘-‘) with
 615  underscores in the controller and method URI segments, thus saving you
 616  additional route entries if you need to do that.
 617  This is required, because the dash isn’t a valid class or method name
 618  character and would cause a fatal error if you try to use it.</p>
 619  </div>
